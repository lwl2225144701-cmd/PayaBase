# 自治 Agent 执行方案

## 范围

这份方案用于把自治 Agent 按最小风险方式落地，不影响现有 RAG、PPT、PDF 和普通聊天链路。

## 执行设计

### 1. 核心入口

在 `chat` 入口里识别“自治任务请求”，命中后交给 `AgentRunner`。

### 2. 执行模型

采用循环式执行：

1. 目标理解
2. 计划生成
3. 工具调用
4. 结果观察
5. 计划调整
6. 完成或终止

### 3. 关键模块

- `core/agent/planner.py`
- `core/agent/runner.py`
- `core/agent/state.py`
- `core/agent/step_executor.py`
- `core/agent/tool_registry.py`

### 4. 数据结构

建议新增：

- `agent_runs`
- `agent_steps`

用于保存：

- 任务状态
- 执行步骤
- 工具结果
- 错误信息
- 审计记录

### 5. 风险控制

必须加这些限制：

- 最大执行步数
- 最大执行时长
- 工具调用白名单
- LLM 调用超时
- 工具失败重试上限

### 6. 渐进式上线

先只支持内部白名单场景，再逐步开放给更多请求类型。

## 里程碑

### 阶段 1

- 接入 `AgentRunner`
- 支持固定 3 到 5 步的简单规划

### 阶段 2

- 接入状态持久化
- 支持中断恢复和失败回收

### 阶段 3

- 接入动态改计划
- 支持更复杂的多工具协作

### 阶段 4

- 补前端执行过程展示
- 补审计和运维监控

## 验收点

- 复杂任务可自动拆解
- 多步流程可执行
- 失败可回收
- 结果可追踪
