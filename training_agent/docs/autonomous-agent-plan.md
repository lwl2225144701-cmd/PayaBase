# 自治 Agent 计划稿

## 目标

把当前“单次路由 + 单链路执行”的聊天模式，升级成“目标驱动 + 多步规划 + 逐步执行 + 动态调整”的自治 Agent。

## 现状判断

当前系统已经有这些能力：

- 请求路由
- RAG 检索
- 内容生成
- PPT 生成
- PDF 生成

但这些能力仍然是“单次触发、单链路返回”，还不是自治 Agent。

## 计划过程

### 1. 保留现有入口

不重写 `chat` 主入口，先把自治能力挂到现有对话流里。

### 2. 新增 Agent 主循环

新增 `AgentRunner`，负责：

- 理解目标
- 生成计划
- 执行步骤
- 观察结果
- 继续规划或终止

### 3. 工具统一化

把现有能力包装成统一工具：

- `rag_qa`
- `document_summary`
- `content_generation`
- `ppt_generation`
- `pdf_generation`

### 4. 引入状态管理

新增运行态和步骤态，至少保存：

- 任务目标
- 当前计划
- 当前步骤
- 已完成步骤
- 中间产物
- 失败原因
- 重试次数

### 5. 增加护栏

必须限制：

- 最大步数
- 总超时
- 工具白名单
- 重试上限
- 低置信度回退

### 6. 前端展示执行过程

前端不能只显示最终回复，还要显示：

- 当前计划
- 已执行步骤
- 正在执行步骤
- 产物
- 失败回退

## 最小落地顺序

1. 先做 `AgentRunner`
2. 再把现有能力统一成工具
3. 再加持久化状态
4. 再加计划修正能力
5. 最后再扩展更多工具和记忆

## 成功标准

- 用户输入一个复杂目标后，系统能自动拆解步骤
- 系统能按步骤执行，并在必要时改计划
- 系统能返回过程和产物，而不只是单轮文本
